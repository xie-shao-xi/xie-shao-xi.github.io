一、初始化hexo

  1. 准备环境
// 官网https://hexo.io/zh-cn

// 下载git
sudo yum install git-core

// 下载nodejs
sudo yum install -y nodejs
  1. 初始化hexo
// 下载安装hexo脚手架
npm install -g hexo-cli

// 下载hexo
npm install hexo

// 初始化项目（项目名blog，可自定义）
hexo init blog

// 进入项目
cd blog

// 下载依赖
npm install

// 本地执行博客
hexo server // 简写命令 hexo s
  1. 成功的话，访问本地4000端口即可

（一）、hexo、nodejs版本冲突

  1. 下载脚手架时报错
报错：Failed at the hexo-util@3.3.0 postinstall script...

原因：centos8.6，执行sudo yum install -y nodejs下载的nodejs版本为10.x，与最新的hexo-cli冲突

尝试解决：根据官网hexo版本和nodejs对应关系，加上hexo-cli发布时间和hexo版本发布时间，推算需要的hexo-cli版本和hexo版本（原先打算使用nvm升级nodejs版本，但服务器下载一直卡住，就只能换方案）

// 下载hexo-cli时，指定版本，成功下载脚手架
npm install -g hexo-cli@xxx
  1. 下载hexo时报错
报错：TypeError: line.matchAll is not a function...

原因：该方法至少node12.x才能运行，即nodejs版本还是太低

尝试解决：删除原有nodejs，下载新的指定版本的（因直接升级失败，且默认下载就是低版本的）

//查找已下载的nodejs文件
dnf list installed | grep nodejs

// 删除，上述命令执行后出现三列，第一列即是删除的包名
sudo dnf remove nodexxx

// 下载指定版本node（14.x即是14版本）
curl -fsSL https://rpm.nodesource.com/setup_14.x | sudo bash -
sudo yum install -y nodejs
